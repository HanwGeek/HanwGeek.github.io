    1.1 计算机系统简介

    1.1.1 计算机系统

    • 硬件
    • 软件
      • 系统软件 管理计算机系统: 语言处理系统 操作系统 服务性程序 数据库管理系统 网络软件
      • 应用软件 按任务需要编制的各种程序

    1.1.2 计算机系统层级结构

    • 抽象(系统复杂性管理方法)

    1.2 计算机的基本组成

    1.2.1 冯 诺依曼计算机的特点

    • 由五大部件组成

    • 指令和数 据以同等地位存于存储器,可按地址寻访

    • 指令和数据以二进制表示

    • 指令由操作码和地址码组成

    • 存储程序

    • 以运算器为中心

    • 输入设备: 讲信息转换成机器能识别的形式

    • 存储器: 存放数据和程序

    • 运算器: 算术运算 & 逻辑运算

    • 控制器: 指挥程序的运行

    • 输出设备: 将运算结果转换为人民熟悉的形式

    1.2.2 计算机硬件框图

    • 以存储器为中心(将不需要运算的数据直接存储)

    • 系统复杂性管理方法2:
      • 层次化: 将系统划分为多个模块和子模块
      • 模块化: 有明确定义的功能和接口
      • 规则性: 模块更容易重用

    1.2.3 计算机的工作步骤

    上机前的准备
    • 建立数学模型
    • 确定计算方法
    • 编写解题步骤
    • 存储器的解题过程
    存储器的基本组成
    • 存储体 — 存储单元 — 存储原件(0/1)
    • 存储单元: 存放一串二进制代码
    • 存储字: 存储单元中二进制代码的组合
    • 存储字长: 存储单元中二进制代码的位数
    • 按地址寻访

    • MAR: 存储器地址寄存器 反映存储单元的个数
    • MDR: 存储器数据寄存器 反映存储字长
    运算器的基本组成及操作过程

    • 加法操作过程

    • 减法操作过程

    • 乘法操作过程

    • 除法操作过程

    控制器的基本结构和功能
    • 解释指令
    • 保证指令按序执行

    完成一条指令:

    • 取指令 PC (PC +1 => PC)
    • 分析指令: 指令寄存器(IR) 存放当前待执行指令
    • 执行指令 CU

    主机完成一条指令的过程

    • 取数指令

      1. PC将取数指令地址存入MAR
      2. 存储器根据MAR中地址访问存储体获得取数指令
      3. 存储器将取数指令存储到MDR中
      4. MDR将取数指令送入IR
      5. CU读取指令操作码部分进行译码操作
      6. IR将指令中的地址部分送入MAR(IR保存了指令,指令的地址,操作数的地址)
      7. MAR将地址送入存储体
      8. 存储体将地址处的数送入MDR
      9. MDR将数送入ACC

    • 存数操作

      1. PC将取数指令地址存入MAR
      2. 存储器根据MAR中地址访问存储体获得存数指令
      3. 存储器将存数指令存储到MDR中
      4. MDR将存数指令送入IR
      5. CU读取指令操作码部分进行译码操作
      6. IR将指令中的地址部分送入MAR
      7. MAR将地址送入存储体
      8. 将ACC数据送入MDR
      9. 将MDR中的数据存入存储体

    1.3 计算机硬件的主要技术指标

    机器字长
    • CPU一次能处理数据的位数与CPU中的寄存器位数有关
    运算速度
    • 主频
    • 核数, 每个核支持的线程数
    • 吉普森法: $T_{\mathrm{M}}=\sum_{i=1}^{n} f_{i} t_{i}$
    • CPI: 执行一条指令所需的始终周期数
    • MIPS: 每秒执行百万条指令
    • FLOPS: 每秒
    存储容量
    • 主存容量:
      • 存储单元个数 x 存储字长: MAR MDR
      • 字节数
    • 辅存容量:
      • 字节数

    3.1 总线的基本概念

    • 总线: 连接各个部件的信息传输线, 是各个部件共享的传输介质
    • 传输方式: 串行 并行
    总线结构

    3.2 总线的分类

    • 片内总线 芯片内部 的总线
    • 系统总线 计算机各部件之间 的信息传输线
      • 数据总线: 双向 与机器字长、存储字长有关
      • 地址总线: 单向 与存储地址、I/O地址有关
      • 控制总线: 有出 有入
    • 通信总线 计算机系统之间 或 见算计系统与其他系统之间
      • 串行
      • 并行

    3.3 总线特性及性能指标

    • 总线特性
      • 机械特性: 尺寸、形状、 管脚数 及 排列顺序

      • 功能特性: 每根传输线的功能 地址 数据 控制
      • 时间特性: 信号与时序的关系
    • 总线性能指标
      • 总线宽度: 数据线的宽度
      • 标准传输率: 每秒传输最大字节数
      • 时钟同步/异步: 同步、不同步
      • 总线复用: 地址线 与 数据线 复用
      • 信号线数: 地址线、数据线和控制线的 总和
      • 总线控制方式: 突发、自动、仲裁、逻辑、计数
      • 其他指标: 负载能力

    3.5 总线控制

    3.5.1 总线判优控制

    基本概念
    • 主设备(模块): 对总线具有控制权

    • 从设备(模块): 响应从主设备发来的总线命令

    • 总线判优控制

      • 集中式: 链式查询 计数器定时查询 独立请求方式
      • 分布式
    链式查询

    • BG构成链式,按顺序逐个查询请求发送方
    • 设备具有优先级
    • 电路设备故障敏感
    • 结构简单, 设备增加容易
    • 可靠性设计比较容易实现
    • 速度较慢,用于简单和嵌入式系统
    计数器定时查询

    • 计数器的值通过设备地址向外传输
    • 通过计数器自定义优先级
    • 少了BG, 多了设备地址线, 宽度为设备地址数
    独立请求方式

    • 每个设备单独拥有BR, BG线
    • 通过排队器灵活设置优先级
    • 用线较多

    3.5.2 总线通信控制

    • 目的: 解决通信双方的协调配合问题
    总线传输周期
    • 申请分配阶段: 主模块申请,总线仲裁决定(总线判优)
    • 寻址阶段: 主模块向从模块 给出地址和命令(地址寻址从模块, 命令控制从模块)
    • 传输阶段: 主模块和从模块 交换数据
    • 结束阶段: 主/从模块 撤销有关信息
    总线通信的四种方式
    • 同步通信: 由统一时标控制数据传输(在固定时间点由时标控制)
    • 异步通信: 采用应答方式
    • 半同步通信: 同步 异步结合
    • 分离式通信: 充分挖掘系统总线每个瞬间的潜力
    同步通信
    • 数据输入
      • 定宽定距的时标
      • 在固定时间点上给出固定的操作

    • 数据输出
      • 定宽定距的时标
      • 按速度慢的设备来设计时标

    异步通信

    • 不互锁: 请求方和应答方不论对方是否收到信号,在一段时间后都会撤销信号
    • 半互锁: 主设备发出请求, 从设备接受请求后发送应答信号, 主设备接收到应答信号后撤销请求信号, 否则保持
    • 全互锁:主设备发出请求, 从设备接受请求后发送应答信号, 主设备接收到应答信号后撤销请求信号, 主设备撤销请求信号后, 从设备撤销应答信号, 否则保持
    半同步通信(同步 异步混合)
    • 同步:
      • 发送方用系统时钟前沿发信号
      • 接收方用系统时钟后沿判断 识别

    • 异步:

      • 运行不同速度的模块和谐工作
      • 增加一条等待响应信号:$\overline{\mathbf{WAIT}}$

    • 数据输入

      • $T_1$: 主模块发送地址
      • $T_2$: 主模块发送命令
      • $T_W$: 当$\overline{\mathbf{WAIT}}$为低电平时,等待一个$T$
      • $T_W$: 当$\overline{\mathbf{WAIT}}$为低电平时,等待一个$T$
      • $…$
      • $T_3$:从模块提取数据
      • $T_4$:从模块撤销数据, 主模块撤销命令

    通信的共同点
    • 一个总线传输周期
      • 主模块发地址 命令 占用总线
      • 从模块准备数据 不占用总线
      • 从模块向主模块发送数据 占用总线
    分离式通信
    • 一个总线传输周期
      • 子周期1: 主模块申请占用总线, 使用完后放弃总线的使用权
      • 子周期2: 从模块申请占用总线, 将信息发送到总线上
    • 特点
      • 各模块都有权申请占用总线
      • 采用同步方式通信, 不等对方回答
      • 各个模块准备数据时, 不占用总线
      • 总线被占用时, 无空闲
      • 充分提高总线的有效占用

    4.2.1 概述

    主存储器基本组成

    4.2.3 随机存取存储器(RAM)

    静态RAM(SRAM)
    • 基本电路
      • 写入分为两个方向, 左方去反, 右方不变

    • 读操作
      • 取反信息无法输出

    • 写操作
      • 输入数据到A, A’

    • 静态RAM芯片 Intel-2114
      • 10条地址线 4条数据线
      • RAM矩阵(64 x 64)读
      • RAM矩阵(64 x 64)写
    动态RAM(DRAM)
    • 基本电路
      • 电容中无电荷为0, 有为1
      • $T_1$,$T_2$,$T_3$为控制管
    • 芯片举例 Intel 1103
      • 读
      • 写
      • 刷新放大器

    • RAM 4116(单管)外特性

      • 读
        • 通过63号行选择,0号列选择器读出存储器的去反值
      • 写
        • 通过63号行选择,0号列选择进入读放大器, 在存储电容中取反值

    • 动态RAM刷新

      • 在一段时间内不对电容再充电, 电容会逐渐漏电. 所以需要进行刷新
      • 刷新与行地址有关. 刷新整行信息
      • 集中刷新 (存取周期为$0.5\mu s$ ) 128 x 128
        • 刷新期间芯片无法使用
      • 分散刷新 (存取周期为$1 \mu s$)
        • $t_M$读写周期, $t_R$进行DRAM行刷新
        • 每行被刷新了15.6次
      • 异步刷新 (分散, 集中相结合)
        • 划分为$15.6 \mu s$等间隔分组

    • 动态RAM和静态RAM比较

    DRAM(主存) SRAM(缓存)
    存储原理 电容 触发器
    集成度 高 低
    芯片引脚 少 多
    功耗 小 大
    价格 低 高
    速度 慢 快
    刷新 有 无

    4.2.6 主存储器校验

    编码的最小距离

    • 任意两组合法代码之间二进制位数的最少差异

    • 编码的纠错, 检错能力与编码的最小距离有关

      • L: 编码最小距离

      • D: 检测错误的位数

      • C: 纠正错误的位数

    汉明码的组成
    • 采用奇偶校验 分组校验

    • 分组是一种非划分方式 异或校验

    • 添加 $2^k \ge n + k + 1$

    • 校验码的位置: $2^i$位

    • 检测位取值与奇偶校验方法有关

    • 各检测位$C_i$所承担的检测小组为

      • $C_1检测g_1小组: 1, 3, 5, 7, 9 , 11, …$, 位置二进制编码为XXXX1
      • $C_2检测g_2小组: 2, 3, 6, 7, 10, 11,…$, 位置二进制编码为XXX1X
      • $C_3检测g_3小组: 4, 5, 6, 7, 12, 12, …$, 位置二进制编码为XX1XX
      • $C_4检测g_4小组: 4, 5, 6, 7, 12, 13,…$, 位置二进制编码为X1XXX
      • $C_8检测g_5小组: 8, 9, 10, 11, 12, 13, …$,位置二进制编码为1XXXX

    • $g_i$独占检测位$2^{i-1}$位

    • $g_i, g_j$独占$2^{i-1} + 2^{j - 1}$位
    • $g_i, g_j, g_k$独占$2^{i - 1} + 2^{j - 1} + 2^{k - 1}$位

    4.3 高速缓存存储器

    4.3.1 概述

    问题的提出
    • 避免CPU空等现象
    • CPU和主存(DRAM的差异)
    • 程序访问的局部性原理: 时间局部性 空间局部性
    Cache工作原理

    • 主存和缓存的编址

      • 主存和Cache划分为大小相等的块, Cache块数量$C$远远小于主存块数量$M$
      • 标记表示主存块号

    • 命中与未命中

      • 缓存有$C$块, 主存有$M$块, $M >> C$
      • 命中:
        • 主存块调入缓存
        • 主存块与缓存块建立了对应关系
        • 用标记记录与某缓存块建立了对应关系的主存块号
      • 未命中:
        • 主存块未调入缓存
        • 主存块与缓存块未建立对应关系

    • 命中率

      • CPU欲访问的信息在Cache中的比率
      • 命中率与Cache的容量与块长有关
      • 一般每块可取4~8个字
      • 块长取一个存取周期内从主存调出的信息长度

    • Cache-主存系统效率

      • $e = \frac{访问Cache时间}{平均访问时间} \times 100\%$

      • 设命中率为$h$, 访问时间为$t_c$, 访问主存时间为$t_m$

        则$e=\frac{t_{c}}{h \times t_{c}+(1-h) \times t_{m}} \times 100 \%$

      • 访问Cache和主存是并行的

    Cache的基本结构

    • Cache和主存之间以块为单位进行传输
    • 根据特定方法, Cache替换机构将主存中的块替换到Cache中
    • 映射: 决定地址的变换方法
    • 变换: 将主存地址和Cache地址之间做变换
    • 主存先把需要的块送入CPU, 再送入Cache缓存
    Cache的读写操作

    • 读操作

      • 在Cache中查找,如果命中直接从Cache送入CPU
      • 若不命中, 则将主存块送入CPU
      • 同时将主存块送入Cache

    • 写操作 Cache和主存的一致性

      • 写直达法(Write - through)

        • 写操作数据既写入Cache又写入主存
        • 写操作就是访问主存时间
        • Cache块退出时不需要对主存执行写操作, 更新策略容易实现
        • 优点: Cache和主存一直保持一直
        • 缺点: 可能会出现反复写的情况

      • 写回法(Write - back)

        • 写操作只把数据写入Cache而不写入主存
        • 当Cache被替换时才写回主存

        • 在并行计算机上多副本一致性问题

        • 写操作时间是访问Cache时间

        • Cache块退出是, 被替换的块需要写会主存, 增加了Cache复杂性
    Cache的改进
    • 增加Cache的级数
      • 片再(片内Cache)
      • 片外Cache
    • 统一缓存的分立缓存
      • 统一: 数据和指令同时存储
      • 指令Cache 数据Cache分别存储: 与指令执行的控制方式有关 是否流水线

    4.3.2 Cache-主存的地址映射

    直接相联映射
    • 将主存以Cache大小等分为若干区
    • 所有区内编号相等的块共享Cache中同编号的地址
    • 主存字块标记包含区号
    • 结构简单, 但Cache利用率可能较低 $m$块同时抢占同一个Cache块
    全相联映射
    • 只要Cache中存在空闲块, 主存中任何一块都可占用
    • Cache标记需要跟主存中所有块进行比较来寻址
    组相连映射
    • Cache分组, 主存分区, 组数和分区中块数相等
    • 主存中区内编号相同的块可以放入Cache编号对应的组中 (组中不止一块)
    • 按区内编号寻址, 比较组内块标记, 不需要和每个Cache块比较
    映射方法比较
    • 靠近CPU要求速度高, 采用直接相连, 或路数较少的组相连
    • 中间层次 采用组相连
    • 距离CPU最远 采用全相连 提高Cache利用率

    4.3.3 替换算法

    先进先出(FIFO)法
    • 最先放入的块, CPU已经使用过, 在不久的将来不再会使用
    • 无法体现程序的局部性原理
    近期最少使用(LRU)算法

    • 在近阶段使用最少的块

    • 替换掉CPU不再使用或在所有的块中再次使用时间间隔最长

    4.3.4 小结

    • 直接相联
      • 某一 主存块 只能固定 映射到 某一 缓存块
      • 不灵活 Cache利用率低
      • 速度块
    • 全相联
      • 某一 主存块 能 映射到 任一 缓存块
      • 成本高 速度慢
      • Cache利用率较高
    • 组相联
      • 某一 主存块 只能 映射到 某一 缓存 组 中的 任一块
      • 速度快 Cache利用率高

    • 设备选择电路: 确认参与传输设备
      • SEL信号(地址比对)有效 & 启动命令有效 => 接口开始工作
    • D = 0, B = 1: 设备未准备好, 设备忙
    • 启动设备, 输入数据到DBR
    • D = 1, B = 0: 设备准备就绪, 设备空闲 (CPU一直踏步等待)
    • CPU从DBR读入数据

    5.5 程序中断方式

    5.5.1 中断概念

    • CPU运行过程中发生意外时间, 中断当前任务, 转而处理异常情况
    • 处理完成后返回到原程序

    5.5.2 中断的产生

    5.5.3 程序中断方式的接口电路

    中断请求触发器 中断屏蔽触发器

    • D = 1 完成触发器 设备数据准备好 & MASK $\overline{Q}$ => 发出中断请求
    排队器

    • 排队电路相互连接
    • 设备1, 2, 3, 4 优先级降序排列
    • $INTR_i = 1$, $\overline{INTR_i} = 0$, 后继$INTP_i’ = 0$
    中断向量地址形成部件

    • 中断向量: 中断服务程序的中断地址和偏移量, 以及相关状态信息

    • 发生中断时, 一些非体系结构寄存器或表示程序状态的寄存器无法被指令读取

      在计算机内集成为一个字, 称为程序状态字

    • 中断向量地址形成部件:

      • 排队器输出结果作为输入(中断源数量)
      • 输出中断向量地址(中断程序入口地址 或 跳转指令地址)
      • 硬件上视为设备编码器

    5.5.4 I/O中断处理过程

    CPU响应中断条件和时间
    • 条件:
      • 用开中断指令允许中断触发器 EINT = 1
      • 用关中断 指令 将EINT = 0或硬件自动复位
    • 时间:
      • D = 1 & MASK = 0
      • 在每条指令执行阶段结束前, CPU发中断查询信号(INTR = 1)
    处理过程

    • CPU执行主程序过程中, 执行输入指令. 在地址线上给出设备地址
    • 地址信号进入设备选择电路. 选择对应电路 SEL信号有效
    • CPU给出启动命令 在接口中进行译码
    • 启动命令 & SEL => 触发器D = 0(数据未准备好)和 B = 1(设备忙)
    • 数据准备好, 送入接口中的DBR. D = 1(数据准备好) B = 0(设备空闲)
    • MASK非端 = 1 & D = 1 => INTR(中断请求)
    • CPU发出中断查询 INTR = 1, 排队器开始排队
    • 排队器输出信号, CPU发出中断响应信号 => 形成向量地址
    • CPU中的PC利用向量地址, 取出中断程序服务地址或跳转指令地址

    5.5.5 中断服务程序流程

    中断服务程序的流程
    • 保护现场
      • 程序断点的保护(中断返回后执行的下一条指令, 程序执行状态) 中断隐指令: 硬件自动完成的操作
      • 寄存器内容的保护 进栈指令
    • 中断服务
      • 对不同的 I/O 设备具有不同内容的设备服务
    • 恢复现场
      • 体系结构寄存器内容 出栈指令 (有些保存在内存中 取数指令)
    • 中断返回
      • 中断返回指令
    单重中断和多重中断
    • 单重中断 不允许中断 现行的 中断服务程序
      • 恢复现场后 开中断指令 EINT = 1
    • 多重中断 允许级别更高的中断源中断现行的中断服务程序
      • 保护现场后 开中断 EINT = 1

    5.6 DMA方式

    5.6.1 DMA特点

    DMA和程序中断两种方式的数据通路

    • 中断方式 : CPU必须中断来 完成转存操作
    DMA与主存交换数据的三种方式

    • 停止CPU访问主存

      • 控制简单 (CPU放弃主存和总线控制权, 适合大量数据传输)
      • CPU可以在此期间执行其他指令
      • CPU处于不工作状态或保存状态
      • CPU无法访问主存, 未充分发挥CPU对主存的利用率

    • 周期挪用(周期窃取) 访存周期

      • 在数据准备阶段, DMA接口放弃对内存和总线的控制权
      • CPU不访存: 主存和总线控制器交由DMA
      • CPU正在访存: DMA等待
      • 同时请求访存: DMA优先, CPU将总线控制权交由DMA

    • DMA与CPU 交替访问 实用性不强

      • CPU工作周期
        • $C_1$专供DMA访存
        • $C_2$专供CPU访存
      • 不需要申请建立和归还总线的使用权

    5.6.2 DMA接口的功能和组成

    DMA接口功能

    • 向CPU申请DMA传送

    • 处理总线控制权的专交

    • 管理系统总线, 控制数据传送

    • 确定数据传送的首地址和长度

      修正传送过程中数据地址和长度

    • DMA传送结束时,给出操作完成信号

    DMA接口的组成

    • AR: 地址寄存器 记录数据地址 每完成一次输入修改地址 + 1
    • WC: 字数计数器 计数传输数据量 每完成一次传输 + 1
    • BR: 数据缓冲器
    • DAR: 设备地址寄存器 设备选择电路使用
    • 数据线设置AR, WC, DAR, BR
    • DMA控制逻辑: 控制内部设备协调工作
    • DREQ: 设备请求
    • DACK: 设备应答
    • HRQ: 总线使用请求
    • HLDA: 总线使用应答
    • 中断机构: 数据传输完成后处理后续工作
      • WC = 0 => 向中断机构发送溢出信号
      • 中断触发器 = 1 向CPU发起中断请求
    DMA工作过程

    • DMA传送过程

      • 预处理

        • 数据在传输之前的相应设置
        • 总之DMA控制逻辑传送方向(入/出)
        • 设备地址(DAR)
        • 主存地址(AR)
        • 传送字数(WC)

      • 传送过程

      • 数据传输过程(输入)

        • 设备将数据送入BR, 通过DREQ通知控制逻辑
        • DMA通过HRQ, 申请占用总线控制权
        • CPU通过HLDA放弃总线控制权
        • AR给出地址
        • DMA通过DACK通知设备
        • BR向总线输送数据, AR + 1, WC +１
        • WC判断是否传输完, 向中断机构给出溢出信号 INTR = 1
        • 中断机构发出中断请求

      • 数据传输过程(输出)

        • BR将数据送入设备, BR空. 设备通过DREQ通知控制逻辑
        • DMA通过HRQ, 申请占用总线控制权
        • CPU通过HLDA应答, 放弃总线控制权
        • AR给出地址
        • DMA通过DACK通知设备
        • 总线向BR输送数据, AR + 1, WC +１
        • WC判断是否传输完, 向中断机构给出溢出信号
        • 中断机构发出中断请求 INTR = 1

      • 后处理

        • 检验送入主存的数是否正确
        • 是否继续使用DMA
        • 测试传送过程是否正确, 错则转诊断程序
        • 由中断服务程序完成

    • DMA接口与系统连接方式

      • 具有公共请求线的 DMA 请求
        • 共享一条请求线
        • CPU通过查询接口, 按次序查询
        • 越靠近CPU, 优先级越高
      • 独立的DMA请求
        • 独立请求线
        • 排队在CPU内部完成

    • DMA 方式与程序中断方式的比较

      | | 中断方式 | DMA方式 |
      | :—————: | :—————: | :—————: |
      | 数据传送 | 程序 | 硬件 |
      | 响应时间 | 指令执行结束 | 存取周期结束 |
      | 处理异常情况 | 能 | 不能 |
      | 中断请求 | 传送数据 | 后处理 |
      | 优先级 | 低 | 高 |

    • DMA接口类型

      • 选择型

        • 物理上连接多个设备
        • 逻辑上只连接一个设备(整个传送过程)
        • 同一时刻只能服务于一个设备

      • 多路型

        • 物理上连接多个设备

        • 逻辑上上连接多个设备同时工作(子通道视为小型DMA)

        • 数据准备阶段可以有多个设备同时准备

        • 数据传输阶段只有一个设备与主存发生交互
